VE平台

VE，中文名微昳，是一款轻巧、简单、优雅的VX前端集成开发平台！帮您定制您的前端VX项目。

邂逅

VE平台（中文名：微昳平台） 是科蓝公司VX团队，基于 Atom专门为前端工程师推出的一款跨平台文本编辑器。具有简洁和直观的图形用户界面，并有很多有趣的特点：支持PE-VX，CSS，HTML，JavaScript等网页编程语言。它支持宏，自动完成分屏功能，集成了文件管理器。

为何要使用VE平台

基于Web开发的VE平台编辑器具有很多优点：

  1. 绿色开源 — 软件包绿色免安装，支持开源
  2. 多平台 — 支持MAC/WIN/LINUX(支持源码编译安装,也提供二进制安装包)
  3. 类Sublime — 风格和sublime text极其相似,不管是风格还是操作上,快捷键上一些是通用的,应该借鉴
  4. 采用包管理技术 — 采用了node.js来访问文件系统和包管理。
  5. VX页面自动生成 — 自动生成VX交易功能页面及路由，所见即所得的智能拖拽布局，释放前端工程师的创作潜能!
  6. VX代码补全 — 对VX指令、过滤器、服务、以及VX对象方法进行提示补全，降低VX的学习门槛!
  7. 前后端分离开发 — 集成了nginx服务器智能配置，无缝支持VX前后端分离开发!

下载安装

VE平台绿色版下载：可进入下载页面 http://115.182.90.204:8082/vxDesign/index.html#/app/ClassicalDesk进行下载。直接点击下载 按钮进行下载。直接解压文件即可。

解压完成，目录下生成两个文件：

  • Atom — 编辑器的执行文件
  • .atom — 编辑器的配置文件，包括各种插件等

初识

简洁、易操作的编辑界面，感受编码的便捷。

启动

执行Atom文件夹下的 atom.exe 启动

菜单栏

  • 文件 — 文件的保存打开，项目的保存打开，最后一次的项目加载，关闭及设置中心，以及用户自定义的配置(配置文件，初始化脚本，样式风格，代码片段，快捷键配置文件)等
  • 编辑 — 文件编辑的操作，文件编码格式，及行跳转等
  • 查看 — 全屏，字体大小的缩放等
  • 选择 — 选择行，选择单词等选择操作
  • 查找 — 都是关于查询的 ，跟Sublime text相似，快捷键基本一样
  • 扩展 — 包，就是插件列表
  • 帮助 — 帮助文档，软件更新，协议等
  • VX — VX扩展插件，包括重启atom，拖拽生成页面，服务启动
  • VX-Tools — VX扩展的工具插件，包括欢迎页，API帮助文档等

设置中心

File->Settings 进入设置中心（默认快捷键 ctrl+, ）

左侧有以下几个设置分类：

  • 核心设置 — 全局设置，可以设置文件的编码，菜单栏是否显示，忽略文件，文档缩进，字体大小，项目主目录等，这个比sublime text人性化，简洁明了的配置
  • 编辑器设置 — 编辑设置
  • 系统设置 — 系统右键菜单设置，可配置右键菜单“Open with Atom”
  • 快捷键绑定 — 快捷键配置，默认快捷键都汇总于此了，很方便查询对应的快捷键的功能，也方便修改…人性化
  • 扩展 — 插件管理中心，可以设置插件，删除插件及禁用，无安装功能
  • 主题 — 主题管理中心，可以设置主题(支持鼠标选定，也支持写入配置文件生效)，管理主题(删除及在线下载主题)
  • 更新 — 查询社区包的状态，随时随地的更新已安装的插件
  • 安装 — 目前分为两栏，自上而下，第一部分是搜索(可以搜索社区的插件)，下面一部分会展示目前比较流行的插件(可以直接点击下载使用)
  • 打开插件源码目录 — 配置文件集合目录 ，打开 .atom 文件夹

高度自定义，配置你自己的编辑器。

Snippet - 自定义代码片段

snippet 即代码提示，通过输入关键字生成相应的代码片段，也可以通过快捷方式快速生成常用代码。
许多 Core 包和 Community 包都捆绑了它们自带的代码段，这些代码段只能用于它们自身。例如，language-html 包就为 HTML 语法高亮和语法提供了支持，并且带有很多代码段；类似的还有：language-java、language-javascript、language-php等语言包。

自定义代码片段方法：

  1. 通过 File > Snippets 菜单，打开 snippets.cson 文件。修改之后，只要保存了文件，Atom 就会重新加载这个文件，就可以立即使用了。
  2. 自定义代码段的基本格式（有趣的是，你可以通过输入 snip 快速生成代码段模板） 

#snippets.cson文件
#单snip选择器写法
'.source.js':
     'console.log':
          'prefix': 'log'
          'body': 'console.log(${1:"crash"});$2'

#多snip写法：（相同作用域的snip要放在同一选择器下，选择器只能写一个，不然会覆盖）
#html
'.text.html.basic':
  # A
  'Anchor':
    'prefix': 'a'
    'body': '<a href="${1:#}">$2</a>$0'

  #可以使用 CoffeeScript多行语法的 """ 来创建长模板。
  'Abbreviation':
    'prefix': 'abbr'
    'body': """
    	<div>
    		<abbr title="$1">$2</abbr>$0
    	</div>
    """

语法解释（以上述 console.log 为例）：

  1. '.source.js': 最外边的键是选择器，表明这些代码段在哪些文件中才能生效。要想知道这个键的值，最简单的方法就是：进入代码段语言所对应的语言包中，查看 Scope 字符串，如下图的js语言包的scope就是 source.js 。（注意：键值就是 . + Scope ，并且同一个snippets.cson文件中的选择器不能重复，不然会覆盖）
  2. 'console.log': 这一层的键是代码段名称，用于在代码段提示菜单中添加的描述。你可以自由命名。
  3. 代码段名称之后，便是可以出发代码段的 prefix 和 当代码段被触发时将要插入的代码 body
  4. 在 'body' 中，每一个带有数字的 $ 就是一个 tab 键驻留位置。只要代码段被触发，便可以通过 Tab 键遍历这些 tab 驻留位置
    • 具有相同数字的 tab 驻留位置 将会创建多个光标
    • 上面例子中的 crash 字符串会在开始时被选中

插件开发

通过编写插件可以实现定制化功能。

创建插件骨架

  • 启动编辑器，按下 Cmd+Shift+P（MacOS）或者 Ctrl+Shift+P（Windows/Linux）来打开命令面板Command Palette。
  • 搜索“Package Generator: Generate Package”并点击列表中正确的条目，你会看到一个输入提示，输入软件包的路径及名称，例如：“atom-vx”。
  • 按下回车键来生成这个骨架代码包，它会自动在 Atom 中打开。

以atom-vx插件为例，完成的插件包含如下文件夹：

  • keymaps - 快捷键文件
  • lib - 主文件
  • menus - 菜单目录配置文件
  • node_modules - NodeJS模块
  • snippets - 代码片段设置文件
  • package.json - Atom 软件包的入口文件。Atom 通过 package.json 里的条目设置来找到主文件的位置
  • README.md - 说明文档

完成开发后需要将插件文件夹移动到.atom的packages文件夹中，并进入插件进行apm配置。
